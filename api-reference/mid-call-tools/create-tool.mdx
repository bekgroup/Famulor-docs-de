---
title: "Mid-call Action erstellen"
api: "POST /user/tools"
description: "Ein neues Mid-call Action erstellen"
---

Dieser Endpunkt ermöglicht es Ihnen, ein neues Mid-call Action zu erstellen, das von Ihren KI-Assistenten verwendet werden kann, um während Anrufen mit externen APIs zu interagieren.

### Body-Parameter

<ParamField body="name" type="string" required>
  Tool-Name - darf nur Kleinbuchstaben und Unterstriche enthalten und muss mit einem Buchstaben beginnen (z.B. `get_weather`, `book_appointment`)
</ParamField>

<ParamField body="description" type="string" required>
  Detaillierte Erklärung, wann und wie die KI dieses Tool verwenden soll (max. 255 Zeichen)
</ParamField>

<ParamField body="endpoint" type="string" required>
  Gültige URL des aufzurufenden API-Endpunkts
</ParamField>

<ParamField body="method" type="string" required>
  HTTP-Methode: `GET`, `POST`, `PUT`, `PATCH` oder `DELETE`
</ParamField>

<ParamField body="timeout" type="integer" optional>
  Request-Timeout in Sekunden (1-30, Standard: 10)
</ParamField>

<ParamField body="headers" type="array" optional>
  HTTP-Header, die mit der Anfrage gesendet werden sollen
  <Expandable title="Header-Eigenschaften">
    <ParamField body="name" type="string" required>
      Header-Name
    </ParamField>
    <ParamField body="value" type="string" required>
      Header-Wert
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="schema" type="array" optional>
  Parameter, die die KI aus der Unterhaltung extrahieren und an den Endpunkt senden wird
  <Expandable title="Schema-Eigenschaften">
    <ParamField body="name" type="string" required>
      Parameter-Name (2-32 Zeichen, muss mit einem Buchstaben beginnen, kann Buchstaben und Unterstriche enthalten)
    </ParamField>
    <ParamField body="type" type="string" required>
      Parameter-Typ: `string`, `number` oder `boolean`
    </ParamField>
    <ParamField body="description" type="string" required>
      Beschreibung, um der KI zu helfen, diesen Parameter zu extrahieren (3-255 Zeichen)
    </ParamField>
  </Expandable>
</ParamField>

### Tools zu Assistenten hinzufügen

Nach dem Erstellen eines Tools müssen Sie es einem Assistenten zuweisen, um es während Anrufen zu verwenden. Tools werden über die Assistenten-API verwaltet:

- **[Assistent erstellen](/api-reference/assistants/create)** - Verwenden Sie den `tool_ids`-Parameter, um Tools beim Erstellen eines Assistenten zuzuweisen
- **[Assistent aktualisieren](/api-reference/assistants/update)** - Verwenden Sie den `tool_ids`-Parameter, um Tools bei einem bestehenden Assistenten hinzuzufügen, zu entfernen oder zu ersetzen
