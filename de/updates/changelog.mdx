Neu: **MCP OAuth 2.1 (PKCE)**. Server-URL einfügen, OAuth-Anmeldung im Browser abschließen, [API-Schlüssel](https://app.famulor.de/api-keys) einfügen — fertig. ## 🔌 Externe Tools per MCP anbinden

Dein Assistent kann jetzt über **MCP (Model Context Protocol)** — den offenen Standard für KI-Anbindungen an externe Systeme — auf Tools und Live-Daten zugreifen. Statt Tools einzeln zu bauen, verbindest du ein ganzes Toolset in einem Schritt. Zeig den Assistenten auf einen kompatiblen Server (CRM, interne APIs, Wissens-Tool) — und er kann mitten im Gespräch nachschlagen und handeln: in Sprachanrufen, im Web-Widget und im Chat.

<Frame>
<img src="/images/mcp-servers-mid-call-tools.png" alt="MCP-Server-Seite — externe Tool-Server mit dem Assistenten verbinden" />
</Frame>

- **Server in Minuten hinzufügen, ohne Code** — Server-Adresse einfügen und bei Bedarf einen Auth-Header setzen. Nichts installieren, nichts bauen
- **Tools werden automatisch erkannt** — Sobald verbunden, werden alle Tools des Servers mit Beschreibung und Parametern gelistet — du siehst genau, was der Assistent kann
- **Präzise steuern, was freigegeben wird** — Einzelne Tools pro Server ein- oder ausschalten, damit der Assistent nur die Fähigkeiten bekommt, die du willst
- **Einmal verbinden, überall nutzen** — Einen Server einem Assistenten oder vielen zuweisen — alles von einer zentralen Seite aus
- **Status immer im Blick** — Jeder Server zeigt einen Live-„Verbunden“-Indikator, listet verfügbare Tools und meldet bei Problemen klar (mit Re-Test per Klick)
- **Sicher by Design** — Auth-Tokens verschlüsselt gespeichert; du behältst die Kontrolle, welche Tools und Rechte jeder Assistent nutzen darf

Typische Einsätze: CRM für Kontaktabfragen und Deal-Erstellung mitten im Anruf, eigene Systeme für Live-Bestell- oder Kontodaten, oder ein Dokumentations-Tool für präzise Antworten auf Detailfragen.

## 🛠 Überarbeitete Mid-Call Tools — einfacher bauen, einfacher verstehen

Die Tools, mit denen dein Assistent während eines Gesprächs handelt — Bestellung nachschlagen, SMS senden, Termin buchen, Verfügbarkeit prüfen, Formular absenden — wurden komplett neu gestaltet. Das ganze Setup ist so aufgebaut, dass auch ohne technischen Hintergrund und ohne Code eine funktionierende Aktion entsteht.

- **Visuelle Flow-Karte** — Ein neues Schritt-für-Schritt-Layout zeigt genau, wie eine Aktion funktioniert: von den Infos, die der Assistent sammelt, bis zur Antwort — alles in einer klaren Ansicht
- **„So funktioniert's“ direkt dabei** — Verständliche Erklärungen neben jeder Einstellung — du weißt immer, was eine Aktion tut und wann der Assistent sie nutzt
- **Systemvariablen griffbereit** — Live-Details wie Anrufernummer oder aktuelles Datum direkt in die Aktion einfügen, mit Referenz — kein Raten mehr beim Variablennamen
- **Alles an einem Ort** — Eigene Aktionen und verbundene MCP-Server leben jetzt zusammen auf einer überarbeiteten Tools-Seite

## ⚡ Call Actions mit No-Code-Automationen bauen

Das ist der große Schritt. Für eine eigene Assistenten-Aktion brauchst du kein API und keinen Entwickler mehr. Beim Anlegen einer Aktion wählst du **Automation Platform** — und die Plattform baut die technische Verbindung für dich.

<Frame>
<img src="/images/mid-call-tools-automation-platform.png" alt="Mid-Call Tool mit Automation Platform — verknüpfter Flow wird automatisch erstellt" />
</Frame>

- **Verbindung entsteht automatisch** — „Automation Platform“ wählen, verknüpfte Automation wird sofort erzeugt und verdrahtet. Die Web-Adresse für den Assistenten wird im Hintergrund gesetzt — keine IDs zum Kopieren, Einfügen oder Synchronhalten
- **Logik visuell bauen** — Verknüpfte Automation öffnen und Schritte per Drag-and-drop zusammenstellen: Datensatz nachschlagen, Kalender prüfen, Nachricht senden, mehrere Aktionen hintereinander. Abschluss mit „Return Response“, damit der Assistent das Ergebnis im Anruf bekommt
- **Mehrstufige Workflows ohne Code** — Was früher Custom Code brauchte — mehrere Systeme verketten, Daten transformieren, bedingte Logik — lässt sich jetzt komplett per Klick zusammenbauen
- **Smartes Verknüpfen bestehender Automationen** — Bestehende Automation einfügen, die Plattform erkennt den Workflow selbst und zieht Details automatisch nach — Anbindung in Sekunden

Von „Der Assistent soll X können“ zu einer live, funktionierenden Aktion in Minuten — komplett per Klick.

  ## 🎙 Neuer Voice Picker — die passende Stimme in Sekunden

  Das Voice-Dropdown ist Geschichte. Die Stimmenauswahl öffnet jetzt einen **vollständigen Picker** — gebaut für Hunderte Optionen, ohne endloses Scrollen.

  - **Filter nach Geschlecht, Alter und Akzent** — Katalog eingrenzen, plus Freitextsuche
  - **Provider-Tabs** — Stimmen nach Speech Engine gruppiert; jede Karte zeigt ein Provider-Badge
  - **Sprachbewusst** — Der Picker folgt der Assistenten-Sprache und warnt, bevor du eine inkompatible Stimme wählst
  - **Stimmen klonen vor Ort** — Voice Cloning lebt jetzt im Picker; gelöschte Klone mit sauberer Bestätigung
  - **Smarter Sync** — Stimme wählen setzt automatisch die passende Speech Engine — kein doppeltes Konfigurieren

  ## 🏷 Telefonnummer-Spitznamen

  Telefonnummern können jetzt einen **Spitznamen und eine Farbe** haben — damit „die Support-Leitung" nicht mehr „+1 555 0182 … welche war das nochmal?" heißt.

  - **Überall sichtbar** — Spitznamen erscheinen in der Nummernliste, auf Assistenten-Karten, in Anrufdetails, SMS-Verlauf und überall, wo du eine Nummer wählst (Assistenten-Setup, Warm Transfers, Kampagnen-Pools)
  - **Durchsuchbare Selektoren** — Nummer per Spitzname oder Rufnummer finden
  - **API-Support** — Spitznamen in allen Telefonnummer-Responses; setzbar per API, auch für [SIP-Trunks](/de/api-reference/phone-numbers/update)

  ## 🤝 WhatsApp-Koexistenz — automatischer Human Takeover

  Der größte Quality-of-Life-Sprung im Koexistenz-Modus: Antwortet jemand aus deinem Team direkt in der WhatsApp-Business-App, erkennt die KI das und **tritt für diese Konversation zurück**.

  - **Kein Durcheinander mehr** — Sobald ein Mensch vom Handy übernimmt, antwortet die KI in diesem Chat nicht mehr
  - **Klarer Audit-Trail** — Eine Systemnotiz im Verlauf erklärt, warum die KI pausiert hat
  - **Sprachnotizen & Bilder gefixt** — Audio und Bilder von Kunden kommen im Koexistenz-Modus zuverlässig an
  - **Keine Kosten für Sync-Historie** — Beim Verbinden eines bestehenden WhatsApp-Kontos werden alte Konversationen beim Setup nie abgerechnet

  ## 🔔 Webhook- & Integrations-Upgrades

  Der [Conversation-Ended-Webhook](/de/api-reference/webhooks/conversation-ended) bekommt ein großes Upgrade — für alle, die Automatisierungen auf Chat-Konversationen bauen.

  - **Medien-Anhänge inklusive** — Bilder, Audio und Dateien aus der Konversation sind jetzt Teil des Webhook-Payloads
  - **Conversation ID & Zeitstempel** — Payload mit Conversation ID und Zeitstempeln pro Nachricht (lesbar und Unix) — Transkripte präzise rekonstruierbar
  - **Erneut senden per Klick** — Jede Konversation hat eine Aktion „Webhook erneut senden"; Zustellstatus und Antwort werden gespeichert
  - **Bulk-Aktionen** — Webhooks für viele Konversationen auf einmal neu auswerten oder erneut senden — direkt aus der Konversationsliste
