Lerne wie du Variablen in Famulor nutzt, um Informationen während eines Anrufs abzurufen.

## Was ist eine Variable?

Eine Variable ist ein Datenwert, den du während eines Anrufs speichern und wiederverwenden kannst. Mit Variablen kannst du z.B. den Namen des Anrufers, die Telefonnummer oder KI-generierte Werte festhalten.

## System-Variablen

Systemwerte, die automatisch zum Anrufzeitpunkt befüllt werden. Im **System-Prompt** und in Mid-Call-Tools (URL, Header-Wert, statisches Feld) mit **doppelten geschweiften Klammern** verwenden: `{{variable_name}}`.

### Verfügbare Systemvariablen

Diese werden automatisch zum Anrufzeitpunkt befüllt. In der App zum Kopieren klicken und an der gewünschten Stelle einfügen. Bei Web/Chat sind einige leer.

<ResponseField name="{{customer_phone}}" type="string">
Telefonnummer der Gegenseite (Anrufer bei eingehend, Angerufener bei ausgehend). Leer bei Web/Chat.
</ResponseField>

<ResponseField name="{{assistant_phone}}" type="string">
Eigene Telefonnummer des Assistenten. Leer, wenn keine vorhanden.
</ResponseField>

<ResponseField name="{{assistant_id}}" type="string">
Eindeutige ID des Assistenten.
</ResponseField>

<ResponseField name="{{assistant_name}}" type="string">
Name des Assistenten.
</ResponseField>

<ResponseField name="{{current_date}}" type="string">
Aktuelles Datum in der Assistenten-Zeitzone (z. B. `2026-06-15`).
</ResponseField>

<ResponseField name="{{current_time}}" type="string">
Aktuelle Uhrzeit in der Assistenten-Zeitzone (z. B. `14:30`).
</ResponseField>

### Verwendung im System-Prompt

Systemvariablen im System-Prompt mit doppelten Klammern referenzieren:

```
Heute ist der {{current_date}}. Die Anrufernummer ist {{customer_phone}}.
Du bist {{assistant_name}}.
```

### Verwendung in Mid-Call-Tools

In einer Custom-HTTP-Aktion oder MCP-Tool-Konfiguration Systemvariablen einfügen in:

- die Request-**URL**
- einen **Header**-Wert
- ein **statisches Feld** im Request-Body

Sie werden durch Live-Werte ersetzt, sobald der Assistent die Aktion während des Gesprächs ausführt.

<Note>
Systemvariablen nutzen doppelte Klammern (`{{customer_phone}}`). Eigene Call Variables unten nutzen einfache Klammern (`{customer_name}`).
</Note>

## Call Variables

Call Variables sind benutzerdefinierte Variablen, die du vor dem Anruf definieren und an den Assistenten übergeben kannst. Diese ermöglichen es dir, personalisierte Gespräche zu führen und kontextspezifische Informationen bereitzustellen.

### Variable Configuration

#### Variable Definition

Erstelle Call Variables, um Informationen an deinen Assistenten zu übergeben, bevor der Anruf startet.

**Setup-Prozess:**

1. Definiere Variablennamen und Standardwerte
2. Übergebe Werte beim Erstellen von Anrufen/Leads oder beim Importieren von Clients
3. Variablen sind sofort in Prompts und während des Anrufs verfügbar
4. Nutze sie für Personalisierung und Kontext

**Konfiguration:**

- **Variablenname**: Der Bezeichner (z.B. `customer_name`)
- **Standardwert**: Wird verwendet, wenn kein Wert übergeben wird (z.B. "John")

### Variable Usage

#### Verwendung in System-Prompts

So verwendest du Call Variables in deinem Assistenten:

- Referenziere Variablen mit der Syntax `{variable_name}`
- Personalisiere Gespräche dynamisch
- Stelle kontextspezifische Informationen bereit

**Beispiele:**

- **`{customer_name}`** für Personalisierung ("Hallo {customer_name}")
- **`{email}`** für Kalender-Integrationen
- **`{account_type}`** für maßgeschneiderte Antworten
- **`{company}`** für Geschäftskontext

### Data Sources

Es gibt verschiedene Möglichkeiten, Call Variables zu befüllen:

#### Manuelle Eingabe

- Beim Erstellen einzelner Anrufe
- Über die Kampagnen-Lead-Erstellung

#### Import-Methoden

- **CSV-Import**: Importiere Leads mit Variablen-Spalten
- **API-Integration**: Direkte Integration mit Client-Daten
- **Direkte CRM-Synchronisation**: Verbinde dich mit deinem CRM-System

#### Automatisierungsplattform-Integrationen

- **GoHighLevel (GHL)**: Automatisches Abrufen von Kontaktdaten und benutzerdefinierten Feldern
- **Google Sheets**: Import von Leads mit Variablen aus Tabellenspalten
- **250+ beliebte Tools und Plattformen**: Nutze unsere No-Code-Plattform für Integrationen
- **Webhooks**: Eigene Integrationen über Webhooks für benutzerdefinierte Anwendungsfälle

#### Plattform-Integration

- Automatisch befüllt aus Client-Profilen
- Verbunden mit bestehenden Kundendatenbanken
- No-Code-Automatisierungs-Workflows

## Best Practices

1. **Verwende aussagekräftige Variablennamen**: Nutze klare Namen wie `customer_name` statt `name1`
2. **Definiere Standardwerte**: Stelle sicher, dass wichtige Variablen Standardwerte haben
3. **Dokumentiere deine Variablen**: Halte fest, welche Variablen verfügbar sind und wofür sie verwendet werden
4. **Teste deine Variablen**: Überprüfe, ob Variablen korrekt übergeben und verwendet werden

## Beispiel-Verwendung

### System-Prompt Beispiel

```
Du bist ein Vertriebsassistent für {company}.

Hallo {customer_name}, vielen Dank für dein Interesse an unseren Dienstleistungen.

Basierend auf deinem Account-Typ ({account_type}) kann ich dir folgende Optionen anbieten...
```

### Call-Erstellung mit Variablen

Wenn du einen Anruf erstellst, kannst du Variablen wie folgt übergeben:

```json
{
  "assistant_id": "123",
  "phone_number": "+49123456789",
  "variables": {
    "customer_name": "Max Mustermann",
    "company": "Muster GmbH",
    "account_type": "Premium",
    "email": "max@muster.de"
  }
}
```
