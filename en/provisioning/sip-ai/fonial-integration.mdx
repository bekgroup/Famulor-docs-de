# Fonial Integration

<Note>
  **Fonial supports FQDN.** You can connect your Fonial SIP trunk with Famulor and use your phone numbers with an AI assistant. Follow the step-by-step guide below.
</Note>

The setup is divided into **Famulor** (creating the SIP trunk, SIP address, and allowed IPs) and **Fonial** (creating the SIP user, assigning phone numbers, entering the call destination/FQDN). Technical basics of Fonial (E.164, OPTIONS, IP addresses) can be found in the [General Setup Instructions for Fonial Trunking](https://www.fonial.de/hilfe/trunking/konfiguration-der-telefonanlagen/allgemeine-einrichtungshinweise).

---

## Step 1: Set up in Famulor

1. Log in at [famulor.de](https://famulor.de).
2. Click on **"Your Phone Numbers"** and then **"Integrate SIP Trunk"**.
3. Fill out the **SIP Trunk Configuration**:
   - **Authentication:** Username and password (you get these in Step 2 from your Fonial customer account).
   - **Outgoing Telephony:** SIP server/address and phone number format according to Fonial (E.164 format with “+”).
   - **Incoming Telephony:** Enter the **allowed IP addresses** of Fonial (see [Fonial Fact-Sheet](https://www.fonial.de/hilfe/trunking/konfiguration-der-telefonanlagen/allgemeine-einrichtungshinweise) – section "IP Addresses"). Note the displayed **"Our SIP Address"** – this is needed in Fonial as the FQDN/call destination.
4. Confirm the configuration and create an **AI assistant**, assigning it to the phone number.

<Note>
  The **IP addresses** of Fonial servers can be found in the [Fonial Fact-Sheet](https://www.fonial.de/hilfe/trunking/konfiguration-der-telefonanlagen/allgemeine-einrichtungshinweise) (section 12). These must be entered in Famulor under "Incoming Telephony" as allowed IPs so that incoming calls from Fonial reach Famulor.
</Note>

---

## Step 2: Create SIP User in Fonial

The SIP user is the key element for the setup: you will connect Famulor (or your call destination) with the Fonial trunk using this user's data. Without a SIP user, phone numbers cannot be assigned.

**Source:** [Create SIP User](https://www.fonial.de/hilfe/trunking/einrichtung-trunking/sip-benutzer-anlegen) in the Fonial help.

1. In the **Fonial customer account**, go to the **"SIP Users"** menu and click it.

<img src="/images/fonial/fonial-sip-1.png" alt="Fonial: SIP Users Menu" />

2. Click **"Create New SIP User"** above the table on the right.

<img src="/images/fonial/fonial-sip-2.png" alt="Fonial: Create New SIP User" />

3. Enter a **name** for the SIP user (e.g., site name or "Famulor") in the dialog, then click **"Save"**.

<img src="/images/fonial/fonial-sip-3.png" alt="Fonial: Enter and Save SIP User Name" />

4. The new SIP user appears in the table; the status will initially show as **"Offline"** until the SIP data is entered in the peer (e.g., Famulor/FQDN).

<img src="/images/fonial/fonial-sip-4.png" alt="Fonial: SIP User Overview, Offline Status" />

5. Open the **SIP user details** via the user/gear icon to view the **username**, **password**, and **server URL**. Enter this information in Famulor under the SIP trunk configuration (authentication and, if applicable, outgoing telephony).

<img src="/images/fonial/fonial-sip-5.png" alt="Fonial: SIP User Details – Username, Password, Server URL" />

## Step 3: Assign Phone Numbers in Fonial

Phone numbers must be assigned to the created SIP user so that calls to these numbers are routed to your configured call destination (Famulor).

**Source:** [Assign Phone Numbers](https://www.fonial.de/hilfe/trunking/einrichtung-trunking/rufnummernzuweisung) in the Fonial help.

1. In your customer account, go to **"Phone Numbers"**, select the desired number(s) by checking the box, then click **"Assign SIP User"** (or "Remove SIP User Assignment" if you want to change an assignment).

<img src="/images/fonial/fonial-ruf-1.png" alt="Fonial: Select Phone Number and Assign SIP User" />

2. Select the **SIP user** (e.g., the one created in Step 2) from the dropdown and click **"Save"**.

<img src="/images/fonial/fonial-ruf-2.png" alt="Fonial: Select SIP User and Save" />

3. After correctly configuring the peer (Famulor with FQDN/call destination and SIP data), the SIP user's status changes to **"Online"**.

<img src="/images/fonial/fonial-ruf-3.png" alt="Fonial: Successful Assignment, Online Status" />

---

## Step 4: Enter FQDN / Call Destination in Fonial

Enter the **Famulor SIP address** as the **call destination** (FQDN) in Fonial, which you find in Famulor under "Phone Numbers → Integrate SIP Trunk → Incoming Telephony" as **"Our SIP Address"**. Omit the prefix `sip:`, enter only the domain (e.g., `your-id.sip.livekit.cloud`).

Where exactly you enter the FQDN in your Fonial customer account depends on the current interface – if you have questions, please contact [Fonial Support](https://www.fonial.de) or use the [Fonial Trunking Help](https://www.fonial.de/hilfe/trunking/einrichtung-trunking).

---

## Important Notes (Based on the Fonial Fact-Sheet)

- **Phone number format:** Fonial uses **E.164** (international format with "+", e.g., +49 221 66966966). Use E.164 format accordingly in Famulor and your configuration.
- **OPTIONS packets:** The peer (Famulor) must respond to OPTIONS requests from Fonial with **200 OK**; otherwise, the system is considered offline, and calls will not be passed through.
- **Audio codecs:** G.711, G.711a, G.711μ; Fonial recommends G.722. DTMF according to RFC 2833.

More details: [General Setup Instructions for Fonial Trunking](https://www.fonial.de/hilfe/trunking/konfiguration-der-telefonanlagen/allgemeine-einrichtungshinweise).
