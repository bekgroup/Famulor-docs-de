# Connect Starface to Famulor

This guide shows how to connect your **Starface** phone system to **Famulor** using a **SIP trunk**. Your AI assistant answers inbound calls; transfers from Famulor must use **SIP** to avoid call loops.

<Info>
  The Starface line may show as **“not registered”** — that is **expected** for IP/host-based SIP to Famulor and is not an error.
</Info>

## Prerequisites

- Starface system with admin access to **Configuration → Lines (Leitungen)**
- Famulor account
- A **free phone number** not already used elsewhere in Starface
- Your **Famulor SIP address** from the dashboard

---

## Step 1: Create a line in Starface

### 1.1 Add a new line

1. In Starface, open **Configuration → Lines (Leitungen)**.
2. Open the **Leitungen** tab.
3. Click **+ Neue Leitung** (New line) at the bottom left.

<Frame>
  <img
    src="/images/starface/starface-neue-leitung.png"
    alt="Starface configuration: Lines tab and New line button for Famulor SIP trunk"
  />
</Frame>

### 1.2 Find your Famulor SIP address

Use this address for **host** and **fromdomain** in Starface:

1. Open [app.famulor.de/phone-numbers](https://app.famulor.de/phone-numbers?lang=en).
2. Click **+ Integrate SIP trunk** (or open **SIP integration** on an existing number).
3. Under **Inbound settings**, copy **Our SIP address** (for Germany/Europe, usually the **France, Germany, Zurich** region).

<Frame>
  <img
    src="/images/starface/starface-famulor-sip-adresse.png"
    alt="Famulor phone numbers: Our SIP address under inbound settings for Starface host and fromdomain"
  />
</Frame>

<Note>
  In Starface, enter only the hostname (e.g. `yourproject.eu.sip.livekit.cloud`) — **without** a `sip:` prefix.
</Note>

### 1.3 Configure the SIP provider in Starface

1. Select the new line and open the **Provider** sub-tab.
2. Under **SIP configuration**, set:

| Field | Value |
| --- | --- |
| **host** | **Our SIP address** copied from Famulor |
| **fromdomain** | Same value as **host** |
| **transport** | **tcp** |
| **nat** | **Yes** (recommended; see [Troubleshooting](#troubleshooting-no-audio--nat)) |

<Frame>
  <img
    src="/images/starface/starface-provider-sip.png"
    alt="Starface edit provider: host, fromdomain, and transport tcp for Famulor SIP"
  />
</Frame>

<Warning>
  Status **“not registered”** is normal. Famulor does **not** use SIP REGISTER for this setup — connectivity is host/IP-based.
</Warning>

### 1.4 Add a phone number to the line

1. Stay on the line and open the **Rufnummern** (Phone numbers) sub-tab.
2. Add the number as **Einzelrufnummer** (single number) with country code **49** and the correct area code and subscriber number.
3. The number must **not** already be in use on another line or service.

<Frame>
  <img
    src="/images/starface/starface-rufnummern.png"
    alt="Starface line phone numbers tab: add single number for Famulor SIP trunk"
  />
</Frame>

### 1.5 Enable CLIP No Screening

1. In line settings ( **Leitungen** tab, line selected), enable **CLIP No Screening**.
2. Note the **line prefix** (e.g. `**2*` or `**3*`) — you need it for time-based forwarding.

<Frame>
  <img
    src="/images/starface/starface-clip-no-screening.png"
    alt="Starface line prefix and CLIP No Screening enabled for Famulor"
  />
</Frame>

<Warning>
  **Save the line prefix:** Forwarding uses the format `**<line-digit>*<phone-number>` (example: `**2*004915112345678`).
</Warning>

## Step 2: Create the number in Famulor

1. Open [app.famulor.de/phone-numbers](https://app.famulor.de/phone-numbers?lang=en).
2. Click **+ Integrate SIP trunk**.
3. Enter the **same phone number** you configured on the Starface line (E.164, e.g. `+4915112345678`).
4. **SIP address / termination:** only required if Famulor should place **outbound** calls via Starface; otherwise leave blank for inbound-only setups.
5. Save the number and copy **Our SIP address** if needed for Starface routing.

### Use SIP for transfers only

<Warning>
  **Do not use PSTN forwarding from the assistant:** Regular call forwarding over the public phone network can create a **loop** with Starface. Configure all assistant transfers as **SIP transfer**.
</Warning>

1. Open the assistant → **Tools** → **Call transfer**.
2. Enable **(Advanced) Is custom SIP transfer?**
3. Under **Custom SIP transfer**, enter the destination (e.g. `tel:…` or Starface format `**<line-digit>*<phone-number>`).
4. Set **When to transfer** to define when the handoff happens (e.g. when the caller asks for a human).

<Frame>
  <img
    src="/images/starface/starface-famulor-sip-transfer.png"
    alt="Famulor call transfer: custom SIP transfer enabled with destination and transfer condition for Starface"
  />
</Frame>

More detail: [SIP troubleshooting – call transfer](/en/provisioning/sip-ai/sip-troubleshooting).

## Step 3: Assign a Starface user

Starface cannot use the SIP number for calls and rules until it is assigned to a user.

1. Go to **Configuration → Users (Benutzer)** and select the user.
2. Open the **Rufnummern** (Phone numbers) tab.
3. Assign the **external SIP number** from the line (e.g. rule **Normaler Anruf** / normal call).

<Frame>
  <img
    src="/images/starface/starface-benutzer-rufnummer.png"
    alt="Starface user phone numbers: assign SIP number for Famulor"
  />
</Frame>

## Step 4: Set up time-based forwarding

Configure routing (e.g. **Modules → Zeitgesteuerte Umleitung** / time-based forwarding) to send calls to Famulor.

1. **Called number:** the Starface number that should reach Famulor.
2. **Destination type:** **Rufnummer** (phone number).
3. **Number** in this exact format: `**<line-digit>*<phone-number>`  
   Example with prefix `**2*` and number `004915112345678`:  
   `**2*004915112345678`
4. Set time windows as needed (e.g. Mon–Fri 07:00–16:00).

<Frame>
  <img
    src="/images/starface/starface-zeitgesteuerte-umleitung.png"
    alt="Starface time-based forwarding: destination number with line prefix asterisk format"
  />
</Frame>

<Note>
  Digits after `*` use the **international dial string without +** (`00` + country code + number). The prefix comes from step 1.5.
</Note>

---

## Step 5: Assign an assistant and test

1. In Famulor, open **Assistants** and edit the desired assistant.
2. Select the correct **inbound type** and the imported **phone number**.
3. **Save assistant** and place a **test call**.

## Troubleshooting: No audio / NAT

**Symptom:** One-way audio or silence on Starface Cloud or with the assistant.

**Fix:** Enable NAT in Starface so RTP (audio) reaches the correct endpoint.

1. Open **Server configuration → Network**.
2. Under **SIP settings**, set **Behind NAT?** to **yes**.
3. Verify **external address** or use **Enter IP** if auto-detection fails.
4. On the line (**Provider**), set **nat** to **Yes**.

<Frame>
  <img
    src="/images/starface/starface-nat.png"
    alt="Starface server network settings: Behind NAT yes for Famulor audio"
  />
</Frame>
