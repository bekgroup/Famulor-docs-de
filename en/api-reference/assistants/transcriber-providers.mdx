description: "Retrieve available STT (speech-to-text) transcriber providers"
---

This endpoint returns a list of available transcriber providers that can be used when creating or updating assistants. Use the returned `id` as `transcriber_provider_id` in the assistant configuration.

### Query parameters

<ParamField query="language_id" type="integer">
  Filter providers by language support. Only providers that support the given language are returned. Use the [Get Languages](/api-reference/assistants/get-languages) endpoint to fetch available language IDs.
</ParamField>

### Response fields

<ResponseField name="data" type="array">
  <Expandable title="Properties">
    <ResponseField name="id" type="integer">
      Unique identifier of the provider. Use this as `transcriber_provider_id` when creating or updating assistants.
    </ResponseField>

    <ResponseField name="name" type="string">
      Display name of the provider (e.g. "Deepgram", "Azure", "Gladia").
    </ResponseField>

    <ResponseField name="code" type="string">
      Internal code of the provider.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>

### Notes

* If `language_id` is not provided, all providers are returned.
* Not every provider supports all languages – use the `language_id` filter to find compatible providers.
* Transcriber providers are only relevant for the `pipeline` mode (the `multimodal` and `dualplex` modes handle STT internally).
* If you do not set `transcriber_provider_id` when creating an assistant, the default provider for the language is used.
