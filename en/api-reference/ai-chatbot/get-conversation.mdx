---
title: "Get conversation"
api: "GET /conversations/{uuid}"
description: "Retrieve full message history for an existing conversation"
---

Fetch the complete message history of an existing Famulor conversation. Use this to resume a chat, display past messages, or analyze interactions.

### Path Parameters

<ParamField path="uuid" type="string" required>
  UUID of the conversation to retrieve
</ParamField>

### Request Example

<CodeGroup>
```bash cURL
curl -X GET "https://app.famulor.de/api/conversations/7c9e6679-7425-40de-944b-e07fc1f90ae7" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch(
  "https://app.famulor.de/api/conversations/7c9e6679-7425-40de-944b-e07fc1f90ae7",
  {
    headers: { Authorization: "Bearer YOUR_API_KEY" },
  }
);

const data = await response.json();
```

```python Python
import requests

conversation_id = "7c9e6679-7425-40de-944b-e07fc1f90ae7"
response = requests.get(
    f"https://app.famulor.de/api/conversations/{conversation_id}",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    timeout=10,
)

print(response.json())
```
</CodeGroup>

### Response Fields

<ResponseField name="status" type="boolean" required>
  Indicates whether the request succeeded
</ResponseField>

<ResponseField name="history" type="array" required>
  Chronological list of messages (oldest first)
  
  <Expandable title="history items">
    <ResponseField name="role" type="string">
      Message role (`assistant` or `user`)
    </ResponseField>

    <ResponseField name="content" type="string">
      Message text content
    </ResponseField>

    <ResponseField name="function_calls" type="array" optional>
      Function calls triggered by the assistant during this message
    </ResponseField>
  </Expandable>
</ResponseField>

### Use Cases

- Resume conversations: restore history when users return
- Analytics: review dialogues for QA or training
- Integration: sync transcripts to CRM or support tools

### Notes

- History is ordered oldest to newest.
- Function calls appear only when the assistant invoked tools during a message.
- Pair with [`Create Conversation`](/api-reference/ai-chatbot/create-conversation) and [`Send Message`](/api-reference/ai-chatbot/send-conversation) for the full flow.
