# Trello Workflow Guide

This page defines a reliable approach for Trello workflows powered by call outcomes and automation events.

## Core Patterns

1. Create: Open a new card when an issue or request is detected.
2. Update: Move card status based on call outcome.
3. Escalate: Mark blockers and notify the responsible owner.

## Quality Controls

- Status transitions only on explicit triggers
- Mandatory comments for escalations
- SLA checks based on due dates
