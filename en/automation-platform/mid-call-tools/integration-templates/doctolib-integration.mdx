# Doctolib Integration Template

Integrate Germany's leading doctor appointment platform Doctolib directly into your Mid-call Actions. Enable your customers to find doctors, check available appointments, and even book appointments – all while on the phone.

## Overview & Features

<CardGroup cols={2}>
  <Card title="Doctor Search Automation" icon="magnifying-glass">
    - Search for specialists based on location and specialty
    - Filter by insurance type (public/private)
    - Availability within defined timeframes
    - Radius-based search for optimal accessibility
  </Card>
  <Card title="Appointment Management" icon="calendar">
    - Real-time availability of doctor appointments
    - Automatic appointment booking during the call
    - Support for video consultations
    - Full patient data integration
  </Card>

## Setting Up Doctolib API Access

### 1. Doctolib Developer Account

<Steps>
  <Step title="Developer Registration">
    - Visit the [Doctolib Developer Portal](https://developers.doctolib.com)
    - Register as a healthcare integration partner
    - Submit your business license and data protection documentation
  </Step>
  
  <Step title="Obtain API Credentials">
    ```yaml
    After approval, you will receive:
      - Client ID
      - Client Secret  
      - API environment (Sandbox/Production)
      - Rate limits and usage guidelines
    ```
  </Step>
  
  <Step title="OAuth 2.0 Setup">
    ```yaml
    OAuth Configuration:
      Authorization URL: https://api.doctolib.de/oauth/authorize
      Token URL: https://api.doctolib.de/oauth/token
      Scopes: 
        - "read:practitioners"
        - "read:appointments" 
        - "write:appointments"
    ```
  </Step>
</Steps>

## Tool 1: Doctor Search Tool

### Configuration in Famulor Interface

<Tabs>
  <Tab title="Tool Details">
    | Field | Value |
    |-------|-------|
    | **Name*** | `Doctolib Arztsuche` |
    | **Description** | "Searches for available doctors and specialists based on location and specialty" |
    | **Function Name*** | `search_doctolib_practitioners` |
    | **Function Description*** | "Searches for doctors by specialty and location. Use this function when the patient is looking for a doctor nearby." |
    | **HTTP Method** | `GET` |
    | **Timeout (ms)** | `5000` |
    | **Endpoint*** | `https://api.doctolib.de/practitioners/search` |
  </Tab>
  
  <Tab title="Header Configuration">
    ```json
    {
      "Authorization": "Bearer {{DOCTOLIB_API_KEY}}",
      "Content-Type": "application/json",
      "User-Agent": "Famulor-MidCall-Actions/1.0"
    }
    ```
  </Tab>
  
  <Tab title="Query Parameters">
    ```json
    {
      "speciality": "{speciality}",
      "location": "{location}",
      "insurance_type": "{insurance_type}",
      "available_in_days": "{availability_days}",
      "radius_km": "{radius}"
    }
    ```
  </Tab>
</Tabs>

### Parameter Schema

```json
{
  "type": "object",
  "properties": {
    "speciality": {
      "type": "string",
      "description": "Medical specialty (e.g. 'General Medicine', 'Dentist', 'Orthopedics', 'Dermatology')",
      "examples": ["General Medicine", "Dentist", "Orthopedics", "Dermatology", "Gynecology"]
    },
    "location": {
      "type": "string", 
      "description": "Postal code or city for the doctor search"
    },
    "insurance_type": {
      "type": "string",
      "enum": ["public", "private", "both"],
      "description": "Insurance type: 'public' (statutory insured), 'private' (private insured), 'both' (both)"
    },
    "availability_days": {
      "type": "integer",
      "description": "Availability within the next X days",
      "minimum": 1,
      "maximum": 90,
      "default": 14
    },
    "radius": {
      "type": "integer",
      "description": "Search radius in kilometers",
      "minimum": 1,
      "maximum": 100,
      "default": 25
    }
  },
  "required": ["speciality", "location"]
}
```

### Response Mapping

```json
{
  "practitioners": "data.practitioners",
  "total": "data.total_count",
  "searchRadius": "data.search_radius_km"
}
```

## Tool 2: Appointment Slots Inquiry

### Configuration in Famulor Interface

<Tabs>
  <Tab title="Tool Details">
    | Field | Value |
    |-------|-------|
    | **Name*** | `Doctolib Terminslots` |
    | **Description** | "Displays available appointments for a specific doctor" |
    | **Function Name*** | `get_doctolib_slots` |
    | **Function Description*** | "Retrieves available appointment slots. Use this function after a doctor has been selected to show concrete appointment options." |
    | **HTTP Method** | `GET` |
    | **Timeout (ms)** | `5000` |
    | **Endpoint*** | `https://api.doctolib.de/practitioners/{practitioner_id}/availabilities` |


</Tabs>

## Complete Workflow

### Step-by-Step Integration

```mermaid
flowchart TD
    A[Patient calls] --> B{Needs doctor appointment?}
    B -->|Yes| C[Ask for specialty]
    B -->|No| Z[Other service]
    C --> D[Ask for location]
    D --> E[search_doctolib_practitioners]
    E --> F{Doctors found?}
    F -->|Yes| G[Patient selects doctor]
    F -->|No| H[Expand search criteria]
    G --> I[get_doctolib_slots]
    I --> J{Appointments available?}
    J -->|Yes| K[Select appointment]
    J -->|No| L[Alternative appointments/doctors]
    K --> M[Collect patient data]
    M --> N[book_doctolib_appointment]
    N --> O[Confirmation to patient]
    H --> D
    L --> G
```

### Practical Conversation Example

<AccordionGroup>
  <Accordion title="Scenario: Dentist Appointment Booking">
    **Patient**: "I need a dentist appointment in Munich."
    
    **AI Assistant**: "I'd be glad to help. Are you insured publicly or privately?"
    
    **Patient**: "Publicly insured."
    
    **AI**: "Perfect. I’m searching for dentists in Munich for you..."
    
    *→ Tool: `search_doctolib_practitioners`*
    ```json
    {
      "speciality": "Dentist",
      "location": "Munich", 
      "insurance_type": "public",
      "availability_days": 14,
      "radius": 25
    }
    ```
    
    **AI**: "I found 8 dentists. Dr. Schmidt on Maximilianstraße has an appointment available the day after tomorrow. Dr. Müller in Schwabing is available next week as well. Which practice would be more convenient for you?"
    
    **Patient**: "Dr. Schmidt sounds good."
    
    **AI**: "I’m checking available appointments for Dr. Schmidt..."
    
    *→ Tool: `get_doctolib_slots`*
    ```json
    {
      "practitioner_id": "dr_schmidt_12345",
      "start_date": "2024-01-17",
      "end_date": "2024-01-31"
    }
    ```
    
    **AI**: "Dr. Schmidt has the following appointments available: Wednesday 14:30, Thursday 09:15, or Friday 16:00. What works best for you?"
    
    **Patient**: "Thursday 09:15 would be perfect."
    
    **AI**: "Excellent! Now I need your details to complete the booking. What is your full name and birthdate?"
    
    *→ Data collection → Tool: `book_doctolib_appointment`*
    
    **AI**: "Appointment booked! You have an appointment with Dr. Schmidt on Thursday, January 18 at 09:15. You will receive a confirmation via SMS and email."

### Error Handling

<AccordionGroup>
  <Accordion title="Common Error Scenarios">
    **No doctors found (404)**:
    ```yaml
    Fallback: "Unfortunately, I couldn’t find any available doctors in your area. 
              Would you like me to expand the search radius or try another city?"
    
    Next-Action: 
      - Increase search radius
      - Suggest alternative cities
      - Offer other specialties
    ```
  </Accordion>
  
  <Accordion title="Appointment Booking Failed">
    ```yaml
    Possible_Causes:
      - Appointment already taken
      - Incomplete patient data
      - Insurance not accepted
    
    Graceful_Handling:
      "I’m sorry, this appointment is no longer available. 
       I can offer you another available slot..."
    
    Fallback_Actions:
      - Retrieve alternative appointment slots
      - Offer a waiting list
      - Suggest a different doctor
    ```
  </Accordion>
  
  <Accordion title="API Rate Limiting">
    ```yaml
    Rate_Limit_Exceeded (429):
      Fallback: "The doctor search is currently highly loaded. 
                May I try again in a few seconds?"
      
    Retry_Logic:
      - Exponential backoff: 2s, 4s, 8s
      - Max 3 retry attempts
      - After 3 failures, offer manual alternative
    ```
  </Accordion>
</AccordionGroup>
