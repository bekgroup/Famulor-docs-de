> Beheben Sie häufige Web-Widget- und Integrationsprobleme

## Widget erscheint nicht

Prüfen Sie:

- Embed-Code vor `</body>` eingefügt
- Browser-Cache per Hard Refresh geleert
- Test im Inkognito-Fenster
- Tarif enthält Web-Widget-Zugriff
- Keine JavaScript-Fehler in der Browser-Konsole
- Frischen Embed-Code aus Widget-Konfiguration genutzt

## Voice funktioniert nicht

Voice benötigt HTTPS.

Prüfen Sie:

- Website läuft über HTTPS
- Mikrofonberechtigung im Browser erlaubt
- Mikrofon funktioniert in anderen Apps
- Moderner Browser wird verwendet
- VPN/Firewall blockiert WebRTC nicht

## Chat funktioniert nicht

Prüfen Sie:

- Browser-Konsole auf Fehler
- Assistent korrekt konfiguriert
- Assistent funktioniert in [Assistent testen](/de/ai-assistants/testing)
- Seite neu laden

## Probleme mit Pre-Chat Formular

Prüfen Sie:

- Felder in Widget-Einstellungen konfiguriert
- Jedes Feld hat Name und Label
- Einstellungen gespeichert
- Feldnamen passen zu Assistenten-Variablen
- Cache geleert und im Inkognito getestet

## Konfiguration wird nicht aktualisiert

Prüfen Sie:

- Widget-Einstellungen aktiv gespeichert
- Embed-Code nach Änderungen neu kopiert
- Alten Code auf Website ersetzt
- Browsercache per Hard Refresh geleert

## WordPress/CMS Probleme

Prüfen Sie:

- Custom HTML Block/Widget verwendet
- Embed-Code vor `</body>` eingefügt
- Caching-Plugin Cache geleert
- Sicherheitsplugins testweise deaktiviert

## Immer noch Probleme?

1. Konsole prüfen
2. Inkognito testen
3. Anderen Browser testen
4. Assistent im Testmodus prüfen
5. Support mit Screenshots kontaktieren
