## Was ist Twilio-Integration?

Die Twilio-Integration ermöglicht es Ihnen, Ihr Twilio-Telefonsystem mit Famulor zu verbinden. Twilio bietet Elastic SIP Trunking, das eine flexible und skalierbare Lösung für die Telefonie-Integration darstellt.

<Steps>
  <Step title="Bestehende Nummern nutzen">
    Ihre bestehenden Twilio-Telefonnummern mit unseren KI-Assistenten nutzen
  </Step>
  <Step title="Infrastruktur beibehalten">
    Twilio's robuste Infrastruktur und globale Reichweite beibehalten
  </Step>
  <Step title="Anrufe verwalten">
    Anrufe über Ihr Twilio-Setup tätigen und empfangen
  </Step>
  <Step title="Kontrolle behalten">
    Die vollständige Kontrolle über Ihre Nummern behalten
  </Step>
  <Step title="Kosten sparen">
    Von Twilio's wettbewerbsfähigen Tarifen und regionalen Endpunkten profitieren
  </Step>
</Steps>

## Einrichtung Ihrer Twilio-Integration

<Steps>
  <Step title="Elastic SIP Trunking erstellen">
    <CardGroup cols={2}>
      <Card
        title="Trunk erstellen"
        icon="plus-circle"
        color="blue">
        <ol>
          <li>Gehen Sie zu Ihrem Twilio-Dashboard</li>
          <li>Navigieren Sie zu "Elastic SIP Trunking"</li>
          <li>Klicken Sie auf "Create Elastic SIP Trunking"</li>
          <li>Geben Sie dem Trunk einen Namen und aktivieren Sie die allgemeinen Einstellungen</li>
        </ol>
        
        ![Twilio Create Elastic SIP Trunking](/provisioning/sip-ai/twillio-create-elastic-sip1.jpeg)
      </Card>
      <Card
        title="Wichtig"
        icon="alert-triangle"
        color="orange">
        Stellen Sie sicher, dass Sie alle erforderlichen Einstellungen korrekt konfigurieren, bevor Sie fortfahren.
      </Card>
    </CardGroup>
  </Step>

  <Step title="Termination einrichten (für ausgehende Anrufe)">
    <CardGroup cols={2}>
      <Card
        title="Termination SIP URI"
        icon="link"
        color="green">
        - Die Termination SIP URI ist hier wichtig - wir werden sie in späteren Schritten verwenden
        - Verwenden Sie eine lokalisierte Termination URI, die sich in Ihrer Nähe befindet
        - Sie können Ihre lokalisierten URIs in der Twilio-Konsole erweitern und anzeigen
        
        ![Twilio Setup Termination](/provisioning/sip-ai/twillio-setup-termination1.jpeg)
      </Card>
      <Card
        title="IP-Whitelist oder Authentifizierung"
        icon="shield"
        color="orange">
        Für Ihren Elastic SIP Trunk, um unsere ausgehenden Anfragen zu akzeptieren, müssen Sie entweder IP-Adressen whitelisten oder eine Authentifizierung mit Benutzername und Passwort erstellen.
        
        ![Twilio New Access Control](/provisioning/sip-ai/twillio-new-access-control.jpeg)
        
        ![Twilio New Access Control Settings](/provisioning/sip-ai/twillio-new-access-control2.jpeg)
      </Card>
    </CardGroup>
    
    <Note>
      Da der Famulor SIP-Server derzeit keine statische IP hat, müssen Sie, wenn Sie sich für die IP-Route entscheiden, alle IP-Adressen im Bereich whitelisten.
    </Note>
    
    <CodeGroup>
      <CodeGroupItem title="IP-Whitelist">
        ```bash
        54.244.51.0
        54.244.51.1  
        54.244.51.2
        54.244.51.3
        54.172.60.0
        54.172.60.1
        54.172.60.2
        54.172.60.3
        ```
      </CodeGroupItem>
    </CodeGroup>
  </Step>
  
  <Step title="Origination einrichten (für eingehende Anrufe)">
    <Card
      title="Famulor SIP-Server"
      icon="server"
      color="blue">
      Hier geben Sie die Famulor SIP-Server-Adresse als Origination SIP URI an:
      <CodeGroup>
        <CodeGroupItem title="SIP URI">
          ```bash
          sip:ihre-unique-id.sip.livekit.cloud
          ```
        </CodeGroupItem>
      </CodeGroup>
      
      <Note>
        **Wichtig:** Die spezifische Famulor SIP-Adresse finden Sie in Ihrem Famulor-Dashboard unter "Telefonnummern" → "SIP-Trunk integrieren" → "Unsere SIP-Adresse".
      </Note>
      
      ![Twilio Setup Origination](/provisioning/sip-ai/twillio-setup-origination1.jpeg)
    </Card>
  </Step>

  <Step title="Nummern zu Elastic SIP Trunking verschieben">
    <Card
      title="Nummern verwalten"
      icon="phone"
      color="green">
      Sie haben den Elastic SIP Trunk erstellt, jetzt müssen Sie Nummern kaufen oder bestehende Nummern zu diesem Trunk verschieben.
      
      ![Twilio Move Numbers](/provisioning/sip-ai/twillio-move-numbers1.jpeg)
    </Card>
  </Step>
  
  <Step title="Nummern zu Famulor importieren">
    <Card
      title="Import-Prozess"
      icon="download"
      color="blue">
      Jetzt, da die Nummer mit Ihrem Elastic SIP Trunking eingerichtet ist, müssen Sie die Nummer zu Famulor importieren, damit wir wissen, wie wir den Anruf routen können.
      
      <ol>
        <li>Gehen Sie zu Ihrem Famulor-Dashboard</li>
        <li>Navigieren Sie zu "Ihre Telefonnummern"</li>
        <li>Klicken Sie auf "SIP-Trunk integrieren"</li>
      </ol>
    </Card>
  </Step>

  <Step title="Details konfigurieren">
    <CardGroup cols={2}>
      <Card
        title="Integrationstyp Details"
        icon="settings"
        color="purple">
        <CodeGroup>
          <CodeGroupItem title="Beispiel">
            ```bash
            Telefonnummer: z.B. +1 555 0123
            Benutzername: Ihr Erweiterungs-Benutzername
            Passwort: Ihr Erweiterungs-Passwort
            ```
          </CodeGroupItem>
        </CodeGroup>
      </Card>
      <Card
        title="Server-Adresse"
        icon="server"
        color="blue">
        Hier geben Sie die Termination SIP URI an, die Sie in Schritt 1 eingerichtet haben.
        <CodeGroup>
          <CodeGroupItem title="Beispiel">
            ```bash
            Server-Adresse: z.B. xxx.pstn.twilio.com
            ```
          </CodeGroupItem>
        </CodeGroup>
      </Card>
    </CardGroup>
    
    <Card
      title="Autorisierungstyp"
      icon="shield"
      color="orange">
      Wählen Sie hier den Autorisierungstyp aus, den Sie in Schritt 2 gewählt haben.
      
      Wenn Sie den IP-Typ gewählt haben, fügen Sie die IPs hinzu:
      
      <CodeGroup>
        <CodeGroupItem title="IP-Whitelist">
          ```bash
          54.244.51.0
          54.244.51.1  
          54.244.51.2
          54.244.51.3
          54.172.60.0
          54.172.60.1
          54.172.60.2
          54.172.60.3
          ```
        </CodeGroupItem>
      </CodeGroup>
    </Card>
    
    <Card
      title="Land auswählen"
      icon="globe"
      color="green">
      Wählen Sie das Land aus, in dem sich Ihr SIP-Trunk befindet.
    </Card>
  </Step>

  <Step title="Einrichtung verifizieren">
    <Card
      title="Verifikationsschritte"
      icon="check-circle"
      color="green">
      Nach der Konfiguration:
      
      <ol>
        <li>Tätigen Sie einen Testanruf an Ihre Nummer/Erweiterung</li>
        <li>Überprüfen Sie, dass der KI-Assistent antwortet</li>
        <li>Verwenden Sie die Testanruf-Funktion, um ausgehende Anrufe zu verifizieren</li>
      </ol>
    </Card>
  </Step>
</Steps>

## Sicherheitsanforderungen

<Card
  title="SIP-Anmeldedaten Sicherheit"
  icon="shield"
  color="red">
  Ihre SIP-Anmeldedaten müssen sicher sein:
  
  - **Benutzername:** Ihre Erweiterungsnummer oder von Twilio bereitgestellter Benutzername
  - **Passwort muss haben:**
    - 12 oder mehr Zeichen
    - Zahlen
    - Groß- und Kleinbuchstaben
    - Sonderzeichen empfohlen
</Card>

## Häufige Probleme

<CardGroup cols={2}>
  <Card
    title="Ausgehende Anrufe funktionieren nicht"
    icon="phone-x"
    color="red">
    <strong>Nach der Verbindung funktioniert der eingehende Anruf, aber der ausgehende funktioniert nicht?</strong>
    
    <Tabs>
      <TabItem title="Termination SIP URI">
        <ul>
          <li>Wenn es ein Leerzeichen gibt, entfernen Sie es bitte</li>
          <li>Verwenden Sie eine lokalisierte Termination URI, die sich in Ihrer Nähe befindet</li>
          <li>Schauen Sie sich diese Dokumentation an, um eine auszuwählen</li>
        </ul>
      </TabItem>
      <TabItem title="Benutzername & Anmeldedaten">
        <ul>
          <li>Stellen Sie sicher, dass Sie den richtigen Benutzernamen und die richtigen Anmeldedaten eingegeben haben</li>
          <li>Beachten Sie, dass der Benutzername nicht der freundliche Name ist, der in der Anmeldedatenliste angezeigt wird</li>
          <li>Der freundliche Name unterscheidet sich vom Benutzernamen - überprüfen Sie dies doppelt</li>
        </ul>
      </TabItem>
    </Tabs>
    
    ![Twilio Outbound Not Working](/provisioning/sip-ai/twillio-outbound-not-work.png)
  </Card>
  
  <Card
    title="Internationale Länder einrichten"
    icon="globe"
    color="blue">
    <strong>Wie richte ich das Wählen in internationale Länder ein?</strong>
    
    <ol>
      <li>Suchen Sie nach "geo", um die "Voice Geographic Permissions"-Einstellung zu finden</li>
      <li>Wählen Sie "Elastic SIP Trunking" im Selektor aus</li>
      <li>Wählen Sie die Länder aus, in die Sie wählen möchten</li>
    </ol>
    
    ![Twilio Choose Elastic SIP Trunking](/provisioning/sip-ai/twillio-choose-elastic-sip-trunking.png)
    
    ![Twilio Setup Dialing International](/provisioning/sip-ai/twillio-setup-dialing-international.png)
  </Card>
</CardGroup>
