# SIP-Integration

## Übersicht

Verbinden Sie Ihr bestehendes Telefonsystem und Ihre Nummern mit Famulor

Dieser Leitfaden hilft Ihnen dabei, die SIP-Integration zu verstehen und einzurichten. Falls Sie während der Einrichtung Klärung benötigen, steht unser Support-Team zur Verfügung, um Ihnen zu helfen.

## Was ist SIP-Integration?

SIP (Session Initiation Protocol) Integration ermöglicht es Ihnen, Ihr bestehendes Telefonsystem mit Famulor zu verbinden. Stellen Sie sich das als den Bau einer Brücke zwischen Ihrem aktuellen Telefon-Setup und unserer KI-Plattform vor. Das bedeutet, Sie können:

- Ihre bestehenden Telefonnummern mit unseren KI-Assistenten nutzen
- Ihr aktuelles Telefonsystem und Ihren Anbieter beibehalten
- Anrufe über Ihr bestehendes Setup tätigen und empfangen
- Die vollständige Kontrolle über Ihre Nummern behalten
- Carrier-Kosten reduzieren, indem Sie lokale VoIP-Anbieter mit besseren Tarifen nutzen

## Wie es funktioniert

Wenn Sie Ihr Telefonsystem mit Famulor integrieren:

**Eingehende Anrufe:** Anrufe an Ihre Nummern werden über Ihr Telefonsystem zu unseren KI-Assistenten weitergeleitet

**Ausgehende Anrufe:** Unsere KI-Assistenten können Anrufe tätigen, die von Ihren Nummern zu kommen scheinen

**Verwaltung:** Sie verwalten Ihre Nummern weiterhin über Ihren aktuellen Anbieter

**Kosteneffizienz:** Nutzen Sie die wettbewerbsfähigen Tarife Ihres VoIP-Anbieters für die Anrufbeendigung

## Integrationsoptionen

Sie können Ihr Telefonsystem auf zwei Arten mit Famulor verbinden:

### 1. PBX-Erweiterungsintegration

Das ist, als würden Sie unserer KI-Plattform eine "Erweiterung" auf Ihrem Telefonsystem geben:

**Was es ist:** Verbinden Sie eine spezifische Erweiterung von Ihrem PBX/Telefonsystem

**Perfekt für:**
- Testen der Integration
- Weiterleitung spezifischer Erweiterungen an die KI
- Einfache Einrichtung mit bestehenden Telefonsystemen

**Sie benötigen:**
- Ihre Erweiterungsnummer (z.B. 1000)
- Erweiterungs-Benutzername und Passwort
- Ihre PBX-Server-Adresse (z.B. pbx.ihrefirma.com)

**Beispiel aus der Praxis:**

```
Ihr Büro verwendet Erweiterungen 1001-1010 für Mitarbeiter
Sie richten Erweiterung 1011 für den KI-Assistenten ein
Anrufe an 1011 gehen an die KI
Die KI kann Anrufe tätigen, die als Erweiterung 1011 erscheinen
```

### 2. Direkte Nummernintegration

Dies verbindet spezifische Telefonnummern mit unserer Plattform:

**Was es ist:** Leiten Sie eine oder mehrere Ihrer Telefonnummern an unsere KI weiter

**Perfekt für:**
- Dedizierte KI-Telefonleitungen
- Kundenservice-Nummern
- Vertriebs- oder Support-Leitungen

**Sie benötigen:**
- Ihre Telefonnummer (z.B. +49 30 123456)
- SIP-Anmeldedaten von Ihrem Anbieter
- SIP-Server-Adresse des Anbieters

**Beispiel aus der Praxis:**

```
Sie haben eine Support-Nummer: +49 30 123456
Sie verbinden diese Nummer mit unserer KI
Alle Anrufe an +49 30 123456 werden vom KI-Assistenten bearbeitet
Die KI macht ausgehende Anrufe, die +49 30 123456 anzeigen
```

## Authentifizierung & Anrufablauf

### Bidirektionale Konfiguration

Die SIP-Integration verwendet dieselben Anmeldedaten für eingehende und ausgehende Anrufe:

**Eingehende Anrufe (VoIP → Famulor):**

```
Ihr VoIP-System authentifiziert sich mit Benutzername/Passwort
↓
Anrufe erreichen unsere Plattform
↓
KI-Assistent antwortet
```

**Ausgehende Anrufe (Famulor → VoIP):**

```
KI initiiert Anruf
↓
Unsere Plattform authentifiziert sich bei Ihrem VoIP mit denselben Benutzername/Passwort
↓
Anruf erreicht Ziel
```

### Anmeldedaten-Synchronisation

Damit die Integration ordnungsgemäß funktioniert:

- Die Anmeldedaten, die Sie im Famulor-Dashboard festlegen, müssen exakt mit Ihrer VoIP-Konfiguration übereinstimmen
- Dieselbe Benutzername/Passwort-Kombination wird für beide Richtungen verwendet
- Alle Änderungen an den Anmeldedaten müssen in beiden Systemen aktualisiert werden
- Wenn Sie Anmeldedaten in Ihrem VoIP-System ändern, denken Sie daran, sie im Famulor-Dashboard zu aktualisieren (und umgekehrt), um die Verbindung aufrechtzuerhalten

## Einrichtung Ihrer Integration

### Schritt 1: Sammeln Sie Ihre Informationen

**Für PBX-Erweiterung:**

```
Erweiterungsnummer: z.B. 1000
Server-Adresse: z.B. pbx.firma.com
Benutzername: Ihr Erweiterungs-Benutzername
Passwort: Ihr Erweiterungs-Passwort
```

**Für direkte Nummer:**

```
Telefonnummer: z.B. +49 30 123456
SIP-Server: z.B. sip.anbieter.com
Benutzername: Von Ihrer Telefongesellschaft bereitgestellt
Passwort: Von Ihrer Telefongesellschaft bereitgestellt
```

### Schritt 2: Konfiguration im Dashboard

1. Gehen Sie zu Ihrem Famulor-Dashboard
2. Navigieren Sie zu "Ihre Telefonnummern"
3. Klicken Sie auf "SIP-Trunk integrieren"

### Schritt 3: SIP-Trunk-Typ auswählen

Wählen Sie oben im Formular einen dieser Trunk-Typen:

- **SIP-Erweiterung** - Verwenden Sie dies, wenn Sie bereits bestehende SIP-Anmeldedaten (eine Erweiterung) haben, die von Ihrem VoIP-Anbieter bereitgestellt werden
- **Telefonnummer (DID)** - Verwenden Sie dies, wenn Sie eine dedizierte Direct Inward Dial (DID) Nummer anstelle einer generischen SIP-Erweiterung integrieren möchten

**Für PBX-Erweiterung:**

```
Erweiterungsnummer: z.B. 1000
Benutzername: Ihr Erweiterungs-Benutzername
Passwort: Ihr Erweiterungs-Passwort
```

**Für direkte Nummer:**

```
Telefonnummer: z.B. +49 30 123456
Benutzername: Von Ihrer Telefongesellschaft bereitgestellt
Passwort: Von Ihrer Telefongesellschaft bereitgestellt
```

### Schritt 4: Ausgehende Einstellungen

Diese Einstellungen steuern, wie Famulor ausgehende Anrufe über Ihren SIP-Anbieter sendet.

**SIP-Adresse** - Geben Sie die Adresse/den Server des SIP-Trunks ohne den Port ein.

**Ausgehender Proxy (Feste IP)** - Einige SIP-Anbieter verlangen, dass Anrufe von einer einzigen, whitelisteten IP-Adresse ausgehen. Konfigurieren Sie dies wie folgt:

- **Toggle EIN** - Wenn Ihr SIP-Anbieter eine feste Quell-IP verlangt, aktivieren Sie "Ausgehender Proxy kommt von einer festen IP-Adresse". Famulor sendet dann alle ausgehenden Anfragen über eine bekannte statische IP. Verwenden Sie diese Option nur, wenn Ihr Anbieter explizit eine einzige Quell-IP verlangt.
- **Toggle AUS** - Wenn Ihr Anbieter Anrufe von beliebigen IPs akzeptiert, lassen Sie diese Option aus. In diesem Fall sendet Famulor Anrufe von dynamischen IPs.

```
SIP-Adresse: (z.B. pbx.firma.com/sip.anbieter.com)
```

### Schritt 5: Nummernformat-Konfiguration

Dies teilt unserem System mit, wie Telefonnummern beim Tätigen von Anrufen formatiert werden sollen. Wählen Sie basierend auf den Anforderungen Ihres Anbieters:

**International mit +**

```
Format: +[Land][Nummer]
Beispiel: +49 30 123456
SIP-Anfrage: sip:+4930123456@ihr.pbx.com
```

**International ohne +**

```
Format: [Land][Nummer]
Beispiel: 49 30 123456
SIP-Anfrage: sip:4930123456@ihr.pbx.com
```

**Nationales Format**

```
Format: [lokale Nummer]
Beispiel: 030 123456
SIP-Anfrage: sip:030123456@ihr.pbx.com
```

### Schritt 6: Eingehende Einstellungen

Diese Einstellungen bestimmen, wie Famulor eingehende Anrufe von Ihrem SIP-Anbieter empfängt und authentifiziert.

**Unsere SIP-Adresse** - Famulor weist eine eindeutige eingehende SIP-URI zu (z.B. 5wed79aeavr.sip.livekit.cloud). Ihr SIP-Anbieter muss so konfiguriert sein, dass er eingehende Anrufe an diese Adresse sendet, indem er seinen ausgehenden Trunk oder seine Routing-Regel auf diese URI zeigt.

**IP-Adresse (Whitelist-IPs)** - Wenn Ihr SIP-Anbieter Anrufe von einer bekannten Menge öffentlicher IP-Adressen senden kann, wählen Sie "IP-Adresse". Fügen Sie dann jede Anbieter-IP zur Whitelist unter "Eingehende Authentifizierung" hinzu.

**Benutzername und Passwort** - Wenn Ihr Anbieter SIP-Auth unterstützt, wählen Sie "Benutzername und Passwort". Dieselben Anmeldedaten, die Sie unter "Ausgehende Einstellungen" eingegeben haben (Benutzername/Passwort), werden zur Authentifizierung eingehender Anrufe verwendet. Stellen Sie sicher, dass die PBX Ihres Anbieters so konfiguriert ist, dass sie diese Anmeldedaten enthält.

### Schritt 7: Länderauswahl

Wählen Sie das Land aus, in dem Ihr SIP-Trunk bereitgestellt ist.

### Schritt 8: Einrichtung verifizieren

Nach der Konfiguration:

1. Tätigen Sie einen Testanruf an Ihre Nummer/Erweiterung
2. Überprüfen Sie, dass der KI-Assistent antwortet
3. Verwenden Sie die Testanruf-Funktion, um ausgehende Anrufe zu verifizieren

## Sicherheitsanforderungen

Ihre SIP-Anmeldedaten müssen sicher sein:

- **Benutzername:** Ihre Erweiterungsnummer oder vom Anbieter bereitgestellter Benutzername
- **Passwort muss haben:**
  - 12 oder mehr Zeichen
  - Zahlen
  - Groß- und Kleinbuchstaben
  - Sonderzeichen empfohlen

## Häufige Fragen

**Kann ich mehrere Nummern verwenden?**

Ja! Sie können mehrere Erweiterungen oder direkte Nummern mit unserer Plattform integrieren.

**Verliere ich die Kontrolle über meine Nummern?**

Nein, Sie behalten die vollständige Kontrolle über Ihren aktuellen Anbieter. Die Integration stellt nur eine Verbindung zwischen Ihrem System und unserer Plattform her.

**Kann ich meine Nummern weiterhin für reguläre Anrufe verwenden?**

Ja, Sie können Routing-Regeln in Ihrem Telefonsystem einrichten, um zu entscheiden, welche Anrufe an die KI gehen und welche an Ihr reguläres Telefonsystem.

**Was, wenn ich Einstellungen später ändern muss?**

Sie können Ihre Integrations-Einstellungen jederzeit über das Dashboard aktualisieren. Alle Konfigurationsoptionen bleiben nach der ersten Einrichtung zugänglich.

**Wie hilft das bei Anrufkosten?**

Indem Sie Ihren eigenen VoIP-Anbieter nutzen, können Sie von deren lokalen Tarifen und bestehenden Vereinbarungen profitieren, die oft wettbewerbsfähiger sind als Standard-Carrier-Tarife. Dies ist besonders vorteilhaft für internationale Anrufe.
