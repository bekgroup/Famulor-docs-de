# Placetel-Limitationen

<Note>
  Placetel unterstützt derzeit leider keine statischen SIP Trunks für die Integration mit Famulor.
</Note>

## Warum funktioniert die Integration derzeit nicht?

<Card
  title="Aktuelle technische Limitation"
  icon="alert-triangle"
  color="red">
  **Placetel unterstützt derzeit leider keine statischen SIP Trunks.**
  
  Dies bedeutet, dass eine direkte SIP-Integration zwischen Placetel und Famulor derzeit technisch nicht möglich ist.
</Card>

<Card
  title="Direkt beim Anbieter nachfragen"
  icon="phone"
  color="blue">
  **Wichtig:** Fragen Sie direkt bei Placetel nach, ob sie statische SIP Trunks anbieten können!
  
  
    - Viele Anbieter entwickeln ihre Services kontinuierlich weiter
    - Möglicherweise ist die Funktion bereits verfügbar oder in Planung
    - Direkte Nachfrage kann zu individuellen Lösungen führen
  
</Card>

## Was bedeutet das für Sie?

<CardGroup cols={2}>
  <Card
    title="Eingeschränkte Funktionalität"
    icon="phone-x"
    color="orange">
    
      - Keine direkte SIP-Trunk-Integration möglich
      - Keine automatische Weiterleitung von Anrufen an KI-Assistenten
      - Eingeschränkte Automatisierungsmöglichkeiten
    
  </Card>
  
  <Card
    title="Alternative Lösungen"
    icon="lightbulb"
    color="blue">
    
      - Manuelle Weiterleitung von Anrufen
      - Verwendung von Famulor-Nummern
      - Call-Forwarding über Placetel-Dashboard
    
  </Card>
</CardGroup>

## Unsere Empfehlung

<Card
  title="Einfache Alternative"
  icon="star"
  color="green">
  **Sie können direkt eine Nummer bei uns kaufen und nutzen!**
  
  
    - **Inbound:** Leiten Sie einfach Ihre aktuelle Placetel-Nummer an die Famulor-Nummer weiter
    - **Outbound:** Verifizieren Sie die Nummer bei "Telefonnummern" und telefonieren Sie direkt damit raus
  
  
  Diese Lösung bietet Ihnen:
  
  
    - Vollständige KI-Assistenten-Funktionalität
    - Automatisierte Anrufbehandlung
    - Erweiterte Telefonie-Features
    - Nahtlose Integration mit der Famulor-Plattform
    - Einfache Einrichtung über Call-Forwarding
  
</Card>
